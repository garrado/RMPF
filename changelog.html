Histórico de Versões
  • v1.5.0 05/05/2026
    Anexo de PDF nos Lançamentos, Monitoramento de Storage, Correções de Permissão e UX do Fiscal
    • Upload de PDF nos lançamentos — fiscal pode anexar, visualizar e remover um PDF por lançamento em competências abertas; barra de progresso durante o envio; anexo visível na conferência e em meus lançamentos (lancamento.html, meus-lancamentos.html, conferencia.html, js/firestore.js, storage.rules, firestore.rules) — PR #74.
    • Card de monitoramento de Storage no Admin — painel admin.html exibe uso total do Firebase Storage, gráfico de barras por competência e tabela com exclusão individual de meses fechados — PR #73.
    • Exclusão em lote de PDFs por período — novo botão 🗑️ Excluir por Período no Admin permite apagar todos os PDFs de um intervalo de meses com dupla confirmação e relatório de resultado — PR #77.
    • CORS do Firebase Storage configurado — criado cors.json autorizando uploads a partir do GitHub Pages; documentação atualizada com comandos gcloud storage — PRs #75 e #76.
    • Correção de permissões na importação de CSV — campo origem dos documentos corrigido para visa_csv/sim_csv; regra Firestore estendida para permitir re-importação SIM; leitura de app_config/github_token liberada para todos os usuários autenticados — PR #78.
    • Correção de logout indevido ao navegar — requireAuth em guard.js parava de redirecionar para index.html (logout) quando qualquer erro async ocorria numa sessão válida; corrigido para redirecionar ao dashboard.html — PR #79.
    • Página Conferência removida do menu do Fiscal — link 🔍 Conferência ocultado da barra lateral do Fiscal em todas as páginas; acesso direto via URL redireciona para meus-lancamentos.html sem logout — PR #80.
  • v1.4.0 05/05/2026
    Suporte a repositório VISA privado na importação de CSV; token GitHub configurável pelo Admin
    • Token GitHub configurável — novo card 🔑 Token do GitHub no painel Admin (admin.html) permite que o administrador configure e salve um PAT (Fine-Grained Token) para acessar o repositório privado garrado/VISA; token armazenado em app_config/github_token no Firestore — PR #71.
    • Importação de CSV via API autenticada — js/visa-import.js e js/sim-import.js substituem o acesso direto à URL pública do GitHub pelo helper fetchGitHubCSV(), que usa a GitHub Contents API com autenticação Bearer — PR #71.
    • Seed de CNAE autenticado — botão 🔄 Sincronizar CNAE com Firestore em admin.html usa a mesma fetchGitHubCSV para importar data/cnae.csv do repositório privado — PR #71.
    • Correção db_setGitHubToken is not a function — funções db_getGitHubToken, db_setGitHubToken e fetchGitHubCSV não estavam exportadas via window.*; corrigido — PR #72.
  • v1.3.3 03/05/2026
    Correção: dropdown de competência do administrador usa o fiscal mais atrasado como referência
    • Lógica do mês aberto para admin corrigida — getProximaCompetencia em js/firestore.js agora agrupa os fechamentos por fiscal, determina o mais recente de cada um e toma o mínimo (fiscal mais atrasado) como referência, evitando que um único fiscal adiantado avance o mês para todos — PR #69.
  • v1.3.2 02/05/2026 - 11:49
    Fechamento em lote para todos os fiscais; dropdown de competência aponta para o próximo mês aberto
    • Botão "Fechar para Todos os Fiscais" — novo botão bulk-close em fechamento.html; o administrador pode fechar a competência para todos os fiscais de uma vez, com confirmação e relatório de resultado — PR #64.
    • Dropdown de competência aponta para o próximo mês aberto — nova função getProximaCompetencia em js/firestore.js substitui db_getUltimoMesFechado nos dropdowns de todas as páginas, apontando para o mês seguinte ao último fechamento — PRs #65, #66, #67.
  • v1.3.1 02/05/2026 - 10:49
    Dropdown de competência padrão para último mês fechado; bloqueio de exclusão em meses fechados
    • Dropdown padrão para último mês fechado — ao abrir qualquer página com seletor de competência (Dashboard, Lançar Atividade, Meus Lançamentos, Conferência, Ocorrências, Fechamento), o mês selecionado automaticamente é o último com dados registrados — PR #63.
    • Bloqueio de exclusão em meses fechados — em meus-lancamentos.html, os botões de excluir/retirar lançamentos manuais ficam ocultos quando a competência está fechada — PR #63.
    • Regra Firestore corrigida para deleção de manuais — o próprio fiscal agora tem permissão de apagar seus registros com status rascunho ou enviado — PR #63.
  • v1.3.0 02/05/2026 - 09:35
    Fechamento, Bloqueios, Ciclo de Recusa e Restrições de Perfil
    • Persistência e cálculo do fechamento — ao fechar uma competência, o registro é gravado no Firestore (fechamentos); pontos com status fechado passam a ser contabilizados no cálculo de PP, PN, PV e Apuração — PR #55.
    • Bloqueio de lançamentos e importações em competências fechadas — lancamento.html exibe banner de aviso e desabilita os botões de salvar e importar; meus-lancamentos.html bloqueia importação e criação de atividades; importadores VISA/SIM ignoram novos registros de competências fechadas — PR #56.
    • Ciclo admin↔fiscal para recusa e reenvio de lançamentos — banner de alerta no dashboard do fiscal quando há lançamentos recusados; reimportação de CSV reseta status recusado para enviado; modal "Corrigir e Reenviar" em meus-lancamentos.html para lançamentos manuais recusados — PR #57.
    • Perfil Administrativo restrito ao dashboard — acesso bloqueado a fechamento.html, conferencia.html, ocorrencias.html e changelog.html; visão de apresentação exibida no lugar da tabela de fiscais — PR #58.
  • v1.2.0 28/04/2026 — 11:22 (UTC-3)
    Importação SIM, Card de Enviados e Correções no Dashboard Fiscal
    • Novo módulo de importação SIM (js/sim-import.js): leitura do CSV auditoria.csv do repositório VISA, com lock distribuído próprio (sim_import_locks) e barra de progresso.
    • Regra Firestore adicionada para a coleção sim_import_locks, corrigindo erro de permissão insuficiente ao acionar a importação SIM.
    • Novo card amarelo no dashboard fiscal exibindo o total de pontos com status enviado (aguardando homologação) na competência selecionada.
    • Reordenação dos cards de métricas fiscais e alargamento do card de Apuração Final para melhor aproveitamento do layout em grade.
    • Correção no botão "Apagar Lançamentos": passa a remover apenas registros importados (prefixos visa_ e sim_), preservando os lançamentos manuais dos fiscais.
  • v1.1.3 28/04/2026
    Lock Distribuído na Importação e Deleção de Registros VISA
    • Implementado lock distribuído via Firestore (visa_import_locks) que impede importações VISA simultâneas para o mesmo mês por usuários diferentes.
    • O mesmo lock protege a deleção em massa de lançamentos VISA do mês em conferencia.html, evitando conflitos de operações concorrentes.
    • Locks obsoletos (acima de 3 minutos) são automaticamente liberados como rede de segurança.
  • v1.1.2 28/04/2026
    Barra de Progresso na Importação VISA
    • Barra de progresso adicionada à importação VISA em lancamento.html e meus-lancamentos.html.
